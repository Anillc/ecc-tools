### Data Model

- `CharCore` is the common payload: `input_slew_idx`, `output_slew_idx`, `driven_cap_idx`, `load_cap_idx`, `delay`, `power`, `PatternId`, and `source_boundary_net_switch_power` are initialized together in the constructor and exposed through simple getters (`src/operation/iCTS/source/database/characterization/CharCore.hh:41`, `src/operation/iCTS/source/database/characterization/CharCore.hh:54`, `src/operation/iCTS/source/database/characterization/CharCore.hh:64`).
- `PatternId` carries a domain tag (`kSegmentPattern` or `kTopologyPattern`) plus `local_id`; `pack()` reserves high bits for the domain to avoid cross-domain hash/id collisions (`src/operation/iCTS/source/database/characterization/PatternId.hh:34`, `src/operation/iCTS/source/database/characterization/PatternId.hh:46`, `src/operation/iCTS/source/database/characterization/PatternId.hh:68`).
- `SegmentChar` wraps `CharCore` and adds `length_idx`. The stored electrical indices are lattice bins, with physical values recovered by multiplying by step size (`src/operation/iCTS/source/database/characterization/SegmentChar.hh:37`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:45`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:58`).
- `HTreeTopologyChar` wraps `CharCore` and adds `levels`; `get_leaf_load_cap_idx()` is currently an alias for the core `load_cap_idx`, which is the downstream-most leaf-side capability (`src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:46`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:56`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:61`).
- `UniformValueLattice` represents uniform bins. `sampleValues()` emits bins `1..steps`; `coveringIndex()` uses ceil; `tryObservedIndex()` rejects nonpositive values and values above max (`src/operation/iCTS/source/database/characterization/ValueLattice.hh:47`, `src/operation/iCTS/source/database/characterization/ValueLattice.hh:60`, `src/operation/iCTS/source/database/characterization/ValueLattice.hh:62`, `src/operation/iCTS/source/database/characterization/ValueLattice.hh:70`, `src/operation/iCTS/source/database/characterization/ValueLattice.hh:81`).

### CharBuilder Flow

- `CharBuilder` exposes `init()`, `build()`, raw segment chars, raw buffering patterns, active wire-length/slew/cap lattice configuration, and overflow diagnostics (`src/operation/iCTS/source/module/characterization/CharBuilder.hh:68`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:72`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:80`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:84`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:87`).
- Initialization clears all previous char state, collects/sorts usable buffers, resolves `max_slew`, `max_cap`, and wire-length unit, then builds sweep vectors from the slew and cap lattices (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:415`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:419`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:440`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:441`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:466`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:468`).
- Buffer discovery uses configured buffer masters, resolves output drive cap via output pin cap limit with table-axis fallback, queries input cap and buffer port names, sorts by max cap, and optionally removes near-neighbor buffers by configured redundancy percentage (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:177`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:293`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:302`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:309`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:325`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:329`).
- Max slew resolution prefers explicit `Config.max_buf_tran`, then minimum liberty input pin slew limit, then minimum input-slew table-axis max; unresolved limits cause the build to skip (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:187`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:189`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:198`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:215`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:559`).
- Max cap resolution similarly prefers explicit `Config.max_cap`, then minimum liberty output pin cap limit, then minimum output-cap table-axis max (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:240`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:242`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:251`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:268`).
- Wire-length unit uses config/caller override first. If absent, it is auto-derived from the strongest usable buffer height times 10 (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:349`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:351`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:360`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:391`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:446`).
- `build()` owns the char-only lifecycle: it calls `STA_ADAPTER_INST.initCharOnly()` before sweep enumeration and `finishCharOnly()` at the end (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:566`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:568`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:654`).
- Per length, topology slots equal the current `length_idx`; all `2^slots` topology bitmasks are enumerated, and each buffer-position topology is expanded into monotonic buffer-master combinations (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:570`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:573`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:687`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:702`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:708`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:714`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:731`).
- Topology bitmasks are pinned to the active wire-length lattice. A set bit creates a buffer position at that lattice slot; terminal slot occupancy at the exact segment end is recorded as terminal branch-buffer metadata (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:147`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:788`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:791`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:801`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:810`).
- `characterizeTopology()` creates a `BufferingPattern` for every topology before feasibility checks. Infeasible patterns are skipped for STA samples but still consume a pattern ID and remain in pattern metadata (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:937`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:941`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:950`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:1086`).
- Pattern feasibility approximates per-stage drive legality before expensive STA. It uses the strongest buffer as the source fixture and weakest buffer as sink fixture; intermediate stages must fit wire cap plus next-buffer input cap within driver max cap, and the last stage computes residual external load capacity (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:827`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:834`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:836`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:851`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:859`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:863`).
- Load points are skipped when the requested load exceeds the residual capacity or when `load_pf - sink_input_cap_pf` is negative. Driven cap is computed as first-buffer input cap plus first wire cap for buffered patterns, or load plus all wire caps for pure-wire patterns (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:968`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:975`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:982`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:984`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:987`).
- Output slew and driven cap must fit the configured lattices. Overflows are counted and samples are dropped (`src/operation/iCTS/source/module/characterization/CharBuilder.cc:877`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:890`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:895`, `src/operation/iCTS/source/module/characterization/CharBuilder.cc:1058`).

### Segment Concatenation

- Segment join condition is exact on electrical boundary bins: upstream `output_slew_idx` equals downstream `input_slew_idx`, and upstream `load_cap_idx` equals downstream `driven_cap_idx`. This is encoded by `SegmentTraits::probeKey()` and `buildKey()` (`src/operation/iCTS/source/module/characterization/SegmentTraits.hh:34`, `src/operation/iCTS/source/module/characterization/SegmentTraits.hh:45`, `src/operation/iCTS/source/module/characterization/SegmentTraits.hh:52`).
- Segment composition keeps the upstream input/driven-cap boundary, downstream output/load boundary, sums length and delay, and computes power as `up.power + down.power - down.source_boundary_net_switch_power`; the upstream source-boundary switch power is preserved (`src/operation/iCTS/source/database/characterization/SegmentChar.hh:63`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:77`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:79`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:83`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:84`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:86`).
- `HashJoinConcat()` builds a hash index over downstream entries using `Traits::buildKey()`, probes each upstream entry with `Traits::probeKey()`, optionally checks `combiner.canCompose()`, obtains a merged `PatternId` from the combiner, and then calls `Traits::compose()` (`src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:95`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:119`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:126`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:133`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:135`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:138`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:139`).
- `SegmentCharTable::concatWith()` is just a facade over `HashJoinConcat<SegmentChar, SegmentTraits>()`, with optional pruner pointer (`src/operation/iCTS/source/module/characterization/SegmentCharTable.hh:83`, `src/operation/iCTS/source/module/characterization/SegmentCharTable.hh:87`).
- `BufferingPattern::concat()` renormalizes upstream buffer positions into the upstream fraction of combined length, offsets downstream positions, appends cell masters, takes terminal branch-buffer semantics from the downstream pattern, and composes monotonic boundary state (`src/operation/iCTS/source/database/characterization/BufferingPattern.hh:133`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:140`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:145`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:149`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:154`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:165`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:166`).

### HTree Concatenation

- H-tree join condition is upstream output slew to downstream input slew plus a binary-fanout cap transform: `ceil(upstream.load_cap_idx / 2)` must equal downstream `driven_cap_idx` (`src/operation/iCTS/source/module/characterization/HTreeTraits.hh:35`, `src/operation/iCTS/source/module/characterization/HTreeTraits.hh:43`, `src/operation/iCTS/source/module/characterization/HTreeTraits.hh:50`, `src/operation/iCTS/source/module/characterization/HTreeTraits.hh:57`).
- H-tree composition sums levels and delay, keeps upstream input/driven-cap boundary, downstream output/load boundary, and computes power as `up.power + 2 * (down.power - down.source_boundary_net_switch_power)` to account for the two downstream branches while avoiding double-counting downstream source-boundary switch power (`src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:66`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:83`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:86`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:90`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:91`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:94`).
- `HTreeTopologyCharTable::concatWith()` is the H-tree table facade over `HashJoinConcat<HTreeTopologyChar, HTreeTraits>()` (`src/operation/iCTS/source/module/characterization/HTreeTopologyCharTable.hh:83`, `src/operation/iCTS/source/module/characterization/HTreeTopologyCharTable.hh:88`).
- `HTreeTopologyPattern` stores only per-level segment pattern IDs rather than expanding the symmetric tree; its `concat()` appends level-segment IDs and sums levels (`src/operation/iCTS/source/database/characterization/HTreeTopologyPattern.hh:35`, `src/operation/iCTS/source/database/characterization/HTreeTopologyPattern.hh:45`, `src/operation/iCTS/source/database/characterization/HTreeTopologyPattern.hh:65`, `src/operation/iCTS/source/database/characterization/HTreeTopologyPattern.hh:68`, `src/operation/iCTS/source/database/characterization/HTreeTopologyPattern.hh:73`).
- In the actual H-tree flow, topology pattern metadata is held in `TopologyPatternRegistry` nodes, and `materialize()` reconstructs the selected per-level segment pattern list from seed/concat nodes (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:416`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:418`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:433`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:470`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:487`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:492`).

### Pruning and Frontier

- The frontier state keys intentionally include electrical boundary bins, source-boundary net switch power, terminal semantic, and monotonic boundary state. Segment and H-tree frontiers use different keys; H-tree additionally includes `leaf_load_cap_idx` (`src/operation/iCTS/source/module/characterization/Frontier.hh:65`, `src/operation/iCTS/source/module/characterization/Frontier.hh:96`, `src/operation/iCTS/source/module/characterization/Frontier.hh:216`, `src/operation/iCTS/source/module/characterization/Frontier.hh:235`).
- Cost dominance is delay/power Pareto dominance within the same group; equal cost breaks by lower packed pattern ID (`src/operation/iCTS/source/module/characterization/Frontier.hh:182`, `src/operation/iCTS/source/module/characterization/Frontier.hh:185`, `src/operation/iCTS/source/module/characterization/Frontier.hh:190`, `src/operation/iCTS/source/module/characterization/Frontier.hh:194`).
- `BuildStateFrontierImpl()` groups entries by state key, sorts each group by delay/power/pattern ID, and keeps entries that strictly improve power as delay increases. The result is sorted with segment- or H-tree-specific sort order (`src/operation/iCTS/source/module/characterization/Frontier.hh:255`, `src/operation/iCTS/source/module/characterization/Frontier.hh:259`, `src/operation/iCTS/source/module/characterization/Frontier.hh:270`, `src/operation/iCTS/source/module/characterization/Frontier.hh:280`, `src/operation/iCTS/source/module/characterization/Frontier.hh:285`, `src/operation/iCTS/source/module/characterization/Frontier.hh:290`).
- The pruned `HashJoinConcat()` path maintains grouped frontier vectors while composing, rejects new results dominated by an existing group member, erases entries dominated by the new result, and flattens groups back into the output (`src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:103`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:140`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:148`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:159`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:162`, `src/operation/iCTS/source/module/characterization/HashJoinEngine.hh:166`).
- `HTreeBuilder` splits base segment chars by length into all, branch-buffered, and leaf-unbuffered frontier families, then builds state frontiers for each family (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:985`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:988`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:994`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1003`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1007`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1011`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1015`).
- If an H-tree level needs a length bin that was not directly characterized, `SynthesizeSegmentEntrySets()` recursively splits the required length into smaller available/synthesized lengths, composes frontier sets, and memoizes the closure problem. It prefers feasible solutions with lower total cost, then more frontier entries, then fewer synthesized lengths (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1066`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1087`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1111`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1132`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1141`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1152`).
- H-tree composition walks selected levels from leaves to root. Each level seeds H-tree chars from that level's segment frontier; later levels compose new seed entries as upstream entries against the accumulated downstream frontier (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1695`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1700`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1730`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1739`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1751`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1758`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1765`).
- Actual-load legality is a second-stage filter after composition. It materializes topology pattern metadata to find the bottom-most buffered level, groups real loads from that boundary, checks fanout and exact electrical/routing legality via clustering, caches by signature, and rejects entries whose leaf load cap does not cover the required real load cap (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1355`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1390`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1456`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1501`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1532`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1561`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1563`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1644`).
- Final selection builds a delay/power Pareto front, sorts it by power then delay and tie-break fields, and chooses the lower median entry. Global selection is performed across all evaluated depth candidates after actual-boundary coverage filtering (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:816`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:823`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:875`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1810`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:1826`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2495`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2500`).
- After materialization, `PruneLeafSingleLoadBuffers()` removes redundant inserted buffers whose output net drives exactly one external load and not another inserted buffer, rewiring the upstream net directly to that load and erasing the buffer, pins, and output net from result storage (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2085`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2100`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2105`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2115`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2120`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2137`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:2291`).

### PatternCombiner

- `PatternCombiner.hh` contains simple standalone `SegmentPatternCombiner` and `TopologyPatternCombiner` classes that only allocate new domain-tagged IDs; they ignore input pattern IDs and do not persist pattern structure (`src/operation/iCTS/source/module/characterization/PatternCombiner.hh:37`, `src/operation/iCTS/source/module/characterization/PatternCombiner.hh:40`, `src/operation/iCTS/source/module/characterization/PatternCombiner.hh:42`, `src/operation/iCTS/source/module/characterization/PatternCombiner.hh:55`, `src/operation/iCTS/source/module/characterization/PatternCombiner.hh:60`).
- The production H-tree flow uses registry-backed combiners instead. `SegmentPatternRegistryCombiner` validates monotonic boundary composition, concatenates `BufferingPattern`, assigns the merged ID, and inserts the merged pattern into `BufferPatternRegistry` (`src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:514`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:519`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:528`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:537`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:538`, `src/operation/iCTS/source/flow/htree/HTreeBuilder.cc:539`).

## Caveats / Not Found

- No source files were modified for this research.
- The requested future numerical directories do not currently contain implementation files: `src/operation/iCTS/source/module/numerical_characterization` and `src/operation/iCTS/source/flow/numerical_htree` were not present.
- `PatternCombiner.hh` is not the full production pattern-registry mechanism; it only generates IDs. The production H-tree flow's structural pattern tracking is embedded in `HTreeBuilder.cc` via `BufferPatternRegistry`, `SegmentPatternRegistryCombiner`, `TopologyPatternRegistry`, and `TopologyPatternRegistryCombiner`.
- The existing flow is deeply enumerative at two levels: `CharBuilder` enumerates all topology bitmasks and monotonic buffer combinations for direct segment characterization, and `HTreeBuilder` composes frontiers across level choices. The frontier pruning reduces dominated states but does not remove the need to generate candidates first.
- Power values are optional and can be zeroed when iPA setup/update fails for a topology/sample. The `source_boundary_net_switch_power` field is critical for avoiding boundary switch-power double counting during segment and H-tree composition.
- The current H-tree data model stores leaf load cap as the core `load_cap_idx`; any numerical model that introduces separate root/leaf capacity concepts must preserve compatibility with `get_leaf_load_cap_idx()` and existing selection/filtering logic.
