### Current Pipeline Shape

The runtime path is:

```text
H-tree flow -> CharacterizationLibrary -> CharBuilder -> STAAdapter char-only iSTA/iPA sampling
            -> SegmentChar + BufferingPattern rows -> segment frontiers
            -> HTreeTopologyChar + HTreeTopologyPattern rows -> topology pruning / selection
```

`HTree.cc` generates topology, runs characterization, gets the cached `CharBuilder`, then copies `get_buffering_patterns()` into a `BufferPatternLibrary` and passes `get_segment_chars()` into segment frontier synthesis (`src/operation/iCTS/source/flow/synthesis/htree/HTree.cc:260`, `src/operation/iCTS/source/flow/synthesis/htree/HTree.cc:269`, `src/operation/iCTS/source/flow/synthesis/htree/HTree.cc:294`, `src/operation/iCTS/source/flow/synthesis/htree/HTree.cc:310`). An analytical module can be inserted only if it produces equivalent segment chars and pattern metadata, or if the downstream consumers are explicitly changed.

`RunCharacterizationFlow` adapts the requested topology level lengths into a `CharBuilder::InitOptions` grid, calls `CharacterizationLibrary::ensure`, verifies non-empty segment chars and a positive length step, then stores lattice metadata in `HTree::BuildResult` (`src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:77`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:128`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:137`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:145`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:147`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:153`). `CharacterizationLibrary` caches by exact `InitOptions` fields, rebuilds with `init()` + `build()`, and fails if no reusable segment chars are produced (`src/operation/iCTS/source/flow/synthesis/htree/characterization/library/CharacterizationLibrary.hh:47`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/library/CharacterizationLibrary.hh:54`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/library/CharacterizationLibrary.cc:36`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/library/CharacterizationLibrary.cc:53`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/library/CharacterizationLibrary.cc:60`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/library/CharacterizationLibrary.cc:62`).

### Pattern Enumeration

`CharBuilder::InitOptions` controls wirelength unit/iterations/indices, max slew/cap, buffer types, redundancy filtering, lattice step counts, routing layer, and wire width (`src/operation/iCTS/source/module/characterization/CharBuilder.hh:59`). The builder stores physical sweep vectors for wirelength, slew, and load before discretizing to integer indices (`src/operation/iCTS/source/module/characterization/CharBuilder.hh:183`).

Initialization filters usable buffers by Liberty-derived drive cap and buffer ports, sorts buffers by max-cap drive strength, and optionally removes near-neighbor buffers using `char_buf_redundancy_pct` (`src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:253`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:262`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:269`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:285`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:289`). Max slew and max cap come from init options first, then Liberty port limits, then Liberty table axes; unresolved limits leave empty sample lattices and later skip build (`src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:149`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:161`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:176`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:201`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:213`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:228`).

Wirelength unit is either configured/overridden or derived from strongest buffer height times 10; wirelength indices are normalized/clamped and converted to physical lengths (`src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:309`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:341`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:354`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:408`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:411`, `src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:420`). Slew/load samples are uniform lattice values from index 1 through the configured step count (`src/operation/iCTS/source/module/characterization/CharBuilderConfig.cc:422`, `src/operation/iCTS/source/database/characterization/ValueLattice.hh:81`, `src/operation/iCTS/source/database/characterization/ValueLattice.hh:89`).

For each characterized wirelength, topology slots equal the length index (`src/operation/iCTS/source/module/characterization/CharBuilderBuild.cc:98`, `src/operation/iCTS/source/module/characterization/CharBuilderBuild.cc:101`). Pattern enumeration iterates every bitset over the topology slots (`2^num_slots`) and, for each selected-buffer topology, iterates monotonic nondecreasing buffer-index combinations (`src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:73`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:88`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:94`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:100`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:117`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:121`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:147`). The bit capacity is clamped/fatal-guarded at `uint64_t` width minus one (`src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:43`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:51`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:76`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternEnumeration.cc:90`).

`buildTopologyDesc` converts selected bit positions to ordered wire segment lengths and records whether the terminal lattice point has a branch buffer at normalized position 1.0 (`src/operation/iCTS/source/module/characterization/CharBuilderTopology.cc:37`, `src/operation/iCTS/source/module/characterization/CharBuilderTopology.cc:54`, `src/operation/iCTS/source/module/characterization/CharBuilderTopology.cc:66`, `src/operation/iCTS/source/module/characterization/CharBuilderTopology.cc:75`). `storeBufferingPattern` turns wire-segment boundaries into normalized `(0, 1]` positions, appends 1.0 for terminal branch-buffer semantics when needed, assigns `PatternId::segment(_next_pattern_id)`, and stores the `BufferingPattern` (`src/operation/iCTS/source/module/characterization/CharBuilderPatternStorage.cc:37`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternStorage.cc:40`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternStorage.cc:53`, `src/operation/iCTS/source/module/characterization/CharBuilderPatternStorage.cc:58`).

Important contract: `storeBufferingPattern` runs before feasibility analysis, so the pattern library can contain infeasible patterns with no corresponding `SegmentChar` rows; downstream only reaches patterns referenced by segment/topology chars, but an analytical producer should avoid dangling references from chars to missing patterns (`src/operation/iCTS/source/module/characterization/CharBuilderSampling.cc:51`, `src/operation/iCTS/source/module/characterization/CharBuilderSampling.cc:53`).

### Feasibility and Sampling

Static feasibility checks each stage cap as wire cap plus downstream input cap against the driver buffer max cap. For the last segment, it computes `max_external_load_pf` as the residual capacity available after wire cap and sink-buffer input cap (`src/operation/iCTS/source/module/characterization/CharBuilderFeasibility.cc:44`, `src/operation/iCTS/source/module/characterization/CharBuilderFeasibility.cc:51`, `src/operation/iCTS/source/module/characterization/CharBuilderFeasibility.cc:55`, `src/operation/iCTS/source/module/characterization/CharBuilderFeasibility.cc:66`, `src/operation/iCTS/source/module/characterization/CharBuilderFeasibility.cc:74`, `src/operation/iCTS/source/module/characterization/CharBuilderFeasibility.cc:75`, `src/operation/iCTS/source/module/characterization/CharBuilderFeasibility.cc:78`).

For a feasible topology, the builder creates a synthetic char-only STA circuit, creates a propagated clock on the source output pin, prepares timing context from source input/source output/sink observation pin, and optionally prepares iPA power over selected temporary buffer insts/nets (`src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:49`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:53`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:54`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:55`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:57`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:59`).

For every sampled load cap, it skips loads beyond feasibility max and loads smaller than the sink input cap. Driven cap is the first inserted buffer input cap plus first wire cap when buffers exist, or external load plus all wire caps for pure wire (`src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:66`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:67`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:73`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:80`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:81`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:85`). Driven-cap overflow outside the configured cap lattice skips all slew samples for that load point (`src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:91`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:94`, `src/operation/iCTS/source/module/characterization/CharBuilderStaSampling.cc:95`).

Per load, `setCharParasitics` rebuilds RC trees for every temporary net using STA wire resistance/capacitance, with external load applied only to the last segment (`src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:48`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:106`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:110`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:113`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:114`). It refreshes cached switch-power load if iPA is already ready, otherwise the first power update will construct iPA later (`src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:49`, `src/operation/iCTS/source/database/adapter/sta/STAAdapterCharPower.cc:155`, `src/operation/iCTS/source/database/adapter/sta/STAAdapterCharPower.cc:159`).

For every sampled input slew, first sample uses full timing preparation/update; later samples use incremental prepare/set/update. The builder queries clock arrival as delay, queried sink slew as output slew, and optional iPA total power/source-boundary net switching power (`src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:60`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:61`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:66`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:68`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:72`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:74`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:78`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:81`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:86`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:88`, `src/operation/iCTS/source/module/characterization/CharBuilderSlewSampling.cc:90`).

The temporary circuit consists of strongest source buffer, weakest sink buffer, optional intermediate buffers, one net per wire segment, source output connected to first net, buffers chained across nets, and sink input connected to the last net. Timing observes the last inserted buffer output when a buffer exists, otherwise the sink input (`src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:43`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:50`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:53`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:59`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:63`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:69`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:75`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:85`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:86`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:88`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:92`, `src/operation/iCTS/source/module/characterization/CharBuilderCircuit.cc:93`).

### Segment and Topology Char Tables

Segment composition contract:

- Join key: upstream `(output_slew_idx, load_cap_idx)` equals downstream `(input_slew_idx, driven_cap_idx)` (`src/operation/iCTS/source/module/characterization/SegmentTraits.hh:31`, `src/operation/iCTS/source/module/characterization/SegmentTraits.hh:45`, `src/operation/iCTS/source/module/characterization/SegmentTraits.hh:52`).
- Composition: length adds; input slew/driven cap come from upstream; output slew/load cap come from downstream; delay adds; power is `up.power + down.power - down.source_boundary_net_switch_power`; source-boundary switch power remains upstream's (`src/operation/iCTS/source/database/characterization/SegmentChar.hh:60`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:77`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:79`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:83`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:84`, `src/operation/iCTS/source/database/characterization/SegmentChar.hh:86`).

H-tree composition contract:

- Join key: upstream output slew equals downstream input slew, and `ceil(upstream.load_cap_idx / 2)` equals downstream driven-cap index (`src/operation/iCTS/source/module/characterization/HTreeTraits.hh:32`, `src/operation/iCTS/source/module/characterization/HTreeTraits.hh:43`, `src/operation/iCTS/source/module/characterization/HTreeTraits.hh:50`, `src/operation/iCTS/source/module/characterization/HTreeTraits.hh:57`).

### Contracts For an `analytical_characterization` Module

Minimum output contracts to preserve if the module replaces `CharBuilder` sampling but keeps downstream pruning:

- Provide positive `wirelength_unit_um`, `wirelength_iterations`, `max_slew`, `max_cap`, `slew_steps`, and `cap_steps`, with `UniformValueLattice` behavior equivalent to current `CharBuilder` accessors (`src/operation/iCTS/source/module/characterization/CharBuilder.hh:84`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:87`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:88`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:90`, `src/operation/iCTS/source/module/characterization/CharBuilder.hh:92`).
- Produce non-empty `std::vector<SegmentChar>` rows whose indices are in lattice range and whose `PatternId`s refer to existing `BufferingPattern`s (`src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:147`, `src/operation/iCTS/source/flow/synthesis/htree/characterization/Characterization.cc:148`, `src/operation/iCTS/test/module/characterization/CharacterizationRealTechSmokeTest.cc:80`, `src/operation/iCTS/test/module/characterization/CharacterizationRealTechSmokeTest.cc:83`).
- Preserve `BufferingPattern` metadata: length index, normalized positions, ordered cell masters, terminal branch-buffer flag, and monotonic boundary state either explicitly or derivable via `BufferPatternLibrary` (`src/operation/iCTS/source/database/characterization/BufferingPattern.hh:79`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:93`, `src/operation/iCTS/source/database/characterization/BufferingPattern.hh:104`, `src/operation/iCTS/source/flow/synthesis/htree/segment_pruning/SegmentLibrary.hh:116`).
- Preserve monotonic buffer-strength composition semantics. Downstream rejects non-monotonic strengthening across concat boundaries via `canCompose` (`src/operation/iCTS/source/database/characterization/BufferingPattern.hh:62`, `src/operation/iCTS/source/flow/synthesis/htree/segment_pruning/SegmentLibrary.hh:470`, `src/operation/iCTS/source/flow/synthesis/htree/segment_pruning/SegmentLibrary.hh:484`).
- Preserve terminal branch-buffer semantics independent of build policy; real-tech slow test expects terminal patterns to remain available when the terminal lattice slot is selected (`src/operation/iCTS/test/module/characterization/CharacterizationRealTechSmokeTest.cc:224`, `src/operation/iCTS/test/module/characterization/CharacterizationRealTechSmokeTest.cc:247`, `src/operation/iCTS/test/module/characterization/CharacterizationRealTechSmokeTest.cc:253`, `src/operation/iCTS/test/module/characterization/CharacterizationRealTechSmokeTest.cc:269`).
- Preserve `source_boundary_net_switch_power` semantics if power remains sampled/estimated, because segment and H-tree composition subtract downstream source-boundary switch power to avoid double counting (`src/operation/iCTS/source/database/characterization/SegmentChar.hh:70`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:84`, `src/operation/iCTS/test/module/characterization/SegmentJoinTest.cc:219`, `src/operation/iCTS/test/module/characterization/HTreeJoinTest.cc:109`).
- Preserve H-tree binary-fanout half-cap join behavior or replace `HTreeTraits`/topology pruning together (`src/operation/iCTS/source/module/characterization/HTreeTraits.hh:35`, `src/operation/iCTS/source/module/characterization/HTreeTraits.hh:43`, `src/operation/iCTS/test/module/characterization/HTreeJoinTest.cc:40`, `src/operation/iCTS/test/module/characterization/HTreeJoinTest.cc:70`).
- Preserve root-driver compensation as a separate concern. Current topology entries get compensation after composition via `RootDriverCompensationPass`, not from `CharBuilder` samples (`src/operation/iCTS/source/flow/synthesis/htree/topology_pruning/TopologyPruning.cc:295`, `src/operation/iCTS/source/database/characterization/HTreeTopologyChar.hh:65`, `src/operation/iCTS/source/database/adapter/sta/STAAdapterRootDriverQuery.cc:219`).
- Keep external-tool access inside `STAAdapter` if analytical mode still uses Liberty/iSTA/iPA queries. Module code currently depends only on CTS/database characterization types and adapter facade calls, matching the database spec boundary.

Replacement choices:

- Drop-in producer: expose the same getters as `CharBuilder` or wrap analytical output behind `CharacterizationLibrary::getCharBuilder()`-compatible surface. This minimizes H-tree flow churn.
- Shared library model: introduce a data-only characterization result type containing segment chars, buffering patterns, lattice metadata, and counters, then adapt both existing `CharBuilder` and analytical builder to produce it. This would reduce the current tight coupling to `CharBuilder`, but requires targeted updates in `CharacterizationLibrary`, `RunCharacterizationFlow`, `HTree.cc`, and tests.
- Full topology producer: directly produce `HTreeTopologyChar` and `TopologyPatternLibrary` from an analytical model. This bypasses segment frontier synthesis but must also replace segment-pruning, topology-pruning, materialization, embedding, sink-load-region, and compensation assumptions. It has a much larger blast radius.
