### Public API forward decisions

- The 4 public methods (`run`, `get_root`, `set_root_guide`, `set_pattern`) plus constructors and dtor stay on `BoundSkewTree` as forwarders to `BoundSkewTreeImpl`.
- `kHalfFactor` stays as public constexpr on `BoundSkewTree` for ABI stability with any external user (even though no external user found in grep).

### Naming changes summary

- `BoundSkewTree::*` chapter slicing (6 .cc files with ClassNameTopic.cc pattern) → real component classes in `detail/` subdirectory each with their own `.hh` + `.cc`.
- Components take `BoundSkewTreeImpl&` by reference on construction; access shared state directly (via `friend class` declarations) and cross-component methods through impl accessor methods (`_impl.balanceSolver().xxx()`).
- Public API of `BoundSkewTree` unchanged (4 public methods + 2 ctors + 1 dtor + 1 public constexpr).
- `BoundSkewTree.cc` is now a thin Pimpl bridge (forwards each public method to `_impl->...`).
- `BoundSkewTree.hh` docstring "intentionally exceeds the 300-line guideline" deleted (file is now 65 lines, well under guideline).
