### Component class design (derived from chapter mapping)

| Component class | Methods | Owns | Reason |
|---|---|---|---|
| `CharSetupConfigurator` | `init(InitOptions)` + collectSortedBuffers + resolveMax{Slew,Cap} + resolveWirelengthUnit + resolveRoutingLayer | sweep-grid setup, buffer discovery, schema setup-table emission | Distinct one-off setup pipeline (was `CharBuilderConfig.cc`) |
| `CharBuildOrchestrator` | `build()` | wirelength sweep loop, progress accumulation, schema results-table emission | Top-level driver (was `CharBuilderBuild.cc`) |
| `CharPatternEnumerator` | calcTopologySlotCount, countSelectedSlots, estimatePatternCountPerWirelength, enumerateWirelength, enumerateTopology, getMonotonicComboCount, advanceToNextMonotonic | Bitset → buffer-monotonic-combo enumeration | Pure combinatorial enumeration |
| `CharTopologyPlanner` | buildBufferingPlan | TopologyBits → BufferingPlan with wire segments | Single-method planner (was `CharBuilderTopology.cc`) |
| `CharFeasibilityChecker` | findBufferInfo, analyzePatternFeasibility | Buffer-info lookup + per-topology max-load feasibility check | Read-only feasibility analysis |
| `CharPatternStorage` | storeBufferingPattern | `_buffering_patterns` vector growth | Single-method storage helper (was `CharBuilderPatternStorage.cc`) |
| `CharCircuitBuilder` | createCharCircuit, setCharParasitics, destroyCharCircuit | Per-pattern temporary CTS circuit lifetime in fast_sta | Circuit lifetime (was `CharBuilderCircuit.cc`) |
| `CharStaSampler` | characterizeTopology, sampleFeasibleTopology, sampleLoadSlews, tryMakeStoredSampleIndices | Per-topology STA sampling pipeline (load × slew × FastSTA query → SegmentChar) | Whole STA sampling pipeline; combines `Sampling/StaSampling/SlewSampling/SampleStorage` chapter files since they form a single linear pipeline that always runs together |

8 components total, falling within the "8-10 component" target band in the brief. `CharSetupConfigurator` and `CharBuildOrchestrator` are kept as separate components because they are the two distinct public-API entry points and need access to schema state + the sweep grid setup, but they cleanly delegate to the lower-level enumeration / planner / sampler components.

The 4 STA sampling chapter files (`CharBuilderSampling.cc` → `CharBuilderStaSampling.cc` → `CharBuilderSlewSampling.cc` → `CharBuilderSampleStorage.cc`) are merged into one `CharStaSampler` because they form a single per-topology pipeline that always runs in the same linear sequence; splitting them would create artificial boundaries that mirror the chapter-file boundaries we are trying to eliminate.

### Naming changes summary

- `CharBuilder::*` chapter slicing (11 .cc files with ClassNameTopic.cc pattern) → real component classes in `detail/` subdirectory each with their own `.hh` + `.cc`.
- Components take `CharBuilderImpl&` by reference on construction; access shared state directly (via `friend class` declarations) and cross-component methods through impl accessor methods (`_impl.feasibilityChecker().analyzePatternFeasibility(...)`).
- Public API of `CharBuilder` unchanged (2 main methods + 1 ctor + 1 dtor + 23 inline getters that became out-of-line forwarders + 1 public nested `InitOptions`). CharBuilder picked up move-ctor + move-assign (originally implicit; now explicitly declared and defaulted in `.cc` because the Pimpl unique_ptr suppresses the implicit copies/moves).
- `CharBuilder.cc` is now a thin Pimpl bridge (ctor / dtor / move-ctor / move-assign / init / build / 23 state accessors all forward to `_impl->...`).
- `CharBuilder.hh` is 116 lines, well under the 200-line target.
- `CharBufferInfo` POD moved to `detail/CharBuilderImpl.hh` (only consumers are detail/-side components).
- `namespace icts::char_builder::detail` isolates all private types + components; matches the W3a `icts::bst::detail` and W2 `icts::sta_adapter::detail` patterns.

### Companion update outside module/characterization/

- `flow/synthesis/htree/characterization/library/CharacterizationLibrary.hh` value-holds `CharBuilder _char_builder`. The original CharBuilder defaulted everything implicitly (including copy + move). The new Pimpl CharBuilder is non-copyable, but Flow.cc has two call sites `_char_library = CharacterizationLibrary{}` that require move-assignment. Added explicit `= default` move ops on CharacterizationLibrary (which were previously implicitly suppressed by `~CharacterizationLibrary() = default`) — and an explicit `= delete` on the copy ops — to keep the existing reset call sites compiling.
