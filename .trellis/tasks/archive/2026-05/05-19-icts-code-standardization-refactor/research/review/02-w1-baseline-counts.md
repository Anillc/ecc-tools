## Failure log

No sub-item required rollback. The Detail-removal verification grep would
have failed for W1b-4 if the struct `RootDriverCompensationDetail` had been
left in place (it lives in `RootDriverCompensation.hh`, outside the file we
renamed), so the struct was renamed to `RootDriverCompensationResult` to
satisfy the grep-pass verification.

The QorEvaluationDetail.hh sub-item's task description assumed only one .cc
file in `flow/evaluation/qor/`, but the directory currently has three
(`QorEvaluation.cc`, `QorEvaluationMetrics.cc`, `QorEvaluationRootProbe.cc`)
that all share the helper functions. Per the task's stated alternative
("如有公开类型必须暴露则内联到 QorEvaluation.hh"), the helpers were inlined
into `QorEvaluation.hh` under the existing `qor_evaluation` namespace
instead of being sunk into a single .cc's anonymous namespace.

The W1b-3 split into 3 headers initially failed compilation because the
`using icts::analytical::AnalyticalMetric;` and
`using icts::analytical::AnalyticalModelKey;` declarations in the deleted
`AnalyticalSolverDetail.hh` were not yet carried over; both were added to
`AnalyticalScoredSegment.hh` / `AnalyticalPatternCompose.hh` and the build
recovered immediately.
